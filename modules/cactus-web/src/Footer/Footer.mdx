# Footer

## Best practices

- The footer should be rendered at the bottom of the page.
- The footer should be rendered as a descendent of `ScreenSizeProvider` in order for the links to be displayed properly.
- You can brand the footer using the `src` prop -- this allows you to pass a logo to the footer.

## Basic usage

The `Footer` component provides two sub-components for rendering content and links:
  - `Footer.Main` - Receives any custom content you'd like to render in the footer as children.
  - `Footer.Link` - Receives a `to` prop & renders a link in the lower section of the footer.

### Try It Out

export const code = `
<Footer src={RepayLogo}>
  <Footer.Main>Custom Footer Content</Footer.Main>
  <Footer.Link to="https://google.com">Privacy Policy</Footer.Link>
  <Footer.Link to="https://repay.com">Careers</Footer.Link>
  <Footer.Link to="https://microsoft.com">Contact Us</Footer.Link>
</Footer>
`
